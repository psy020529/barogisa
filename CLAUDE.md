# 바로기사 (Barogisa) — 프로젝트 컨텍스트

> Claude Code가 이 파일을 자동으로 읽는다. 개발 시작 전 반드시 전체를 숙지할 것.

---

## 1. 프로젝트 한 줄 정의

**주방가구 시공기사 · 공장 · 팀장을 연결하는 B2B 운영 SaaS 모바일 앱.**
매일 다른 공장에서 일하는 시공기사의 일감 수락 · 현장 기록 · 청구 · 수금을 하나의 흐름으로 묶는다.

---

## 2. 사업 배경 (왜 이 앱인가)

- 창업자는 주방가구 현장 감리·운영 5년, 누적 시공 중개 200건 경험자
- 기사 20명 인터뷰를 통해 공통 통증 확인:
  - 일감 조율을 전화·카톡으로만 함 → 일정 충돌, 연락 두절
  - 현장 기록이 없어 분쟁 시 근거 없음
  - 월말 청구를 수기로 계산, 미수금 추적 불가
- 기존 플랫폼(숨고·크몽)은 익명 매칭 → 주방가구 시장에서 안 통함 (AS 책임이 거래에 묶여 있어 공장은 검증된 기사만 씀)
- 우리는 **매칭으로 진입하지 않는다** → 이미 일하는 사람들의 운영을 자동화하는 도구로 진입

## 4. 핵심 5단계 흐름 (MVP의 전부)

```
[공장] 일감 등록
    ↓ 푸시 알림
[기사] 한 탭으로 수락/거절 (일정 충돌 시 자동 경고)
    ↓
[기사] 현장 도착 → GPS 자동 체크인 + 작업 시작 사진
[기사] 작업 완료 → 체크아웃 + 완료 사진
    ↓
[앱] 월말 공장별 작업횟수·금액 자동 합산 → 청구서 PDF 자동 생성
    ↓
[공장] 송금 → [기사] 입금 확인 한 탭 매칭
```

## 5. MVP 핵심 화면 (우선순위 순)

1. **캘린더 화면** — 시공기사 메인. 본인 일정(파랑) + 팀원 일정(초록) + 신규 일감(빨강) 색상 점 표시. 상단에 이번달 수익 / 미수금 / 이번주 일감 요약 3칸.
2. **일감 수락/거절 카드** — 공장명, 날짜, 공정, 단가 표시. 한 탭 수락/거절.
3. **체크인/체크아웃** — GPS 자동 인식. 사진 첨부.
4. **정산 화면** — 공장별 작업 내역, 미수금 추적, 청구서 PDF 생성.
5. **공장 측 일감 등록** — 일자·공정·주소·지정기사 입력. 단가 자동 입력(시장 표준).

## 10. 핵심 비즈니스 규칙

- 공장은 기사를 **지명**해서 발주한다 (익명 매칭 없음)
- 양측 수락 후 거래 성립 (공장 발주 → 기사 수락 → 공장 최종 확정)
- 주황 상태 = 기사가 수락했으나 공장이 아직 미확정
- 공장 연락처는 **앱 내에서만 노출** (탈중개 방지, 락인 핵심)
- 팀장이 팀원 등록 시 페이백 지급 (바이럴 유입 전략)

## 11. 개발 시 주의사항

- 모바일 퍼스트 설계. 기사들은 현장에서 한 손으로 조작.
- 핵심 액션(수락/거절/체크인)은 **한 탭**으로 완료 가능해야 함.
- GPS 체크인 마찰 최소화가 핵심 UX 과제.
- 오프라인 상태에서도 체크인 기록이 저장되어야 함 (현장 인터넷 불안정).
- 창업자가 코드를 직접 검토할 수 있도록 복잡한 추상화보다 명확한 코드 우선.
