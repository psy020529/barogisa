# 바로기사 — 개발자 전달 문서 (MVP)

> **범위:** 기록·정산 중심 캘린더 앱. 일감 매칭·공장 계정·블라인드 매칭은 **이번 범위 제외**(Phase 2).
> **1차 사용자:** 시공 팀장 (단일 역할). 팀원은 가입 없이 팀장이 이름만 등록(라이트).
> **플랫폼:** 모바일 우선. 와이어프레임: `baro-gisa-wireframe.html` 참조.

---

## 1. 화면 구성 & 내비게이션 맵

```
[하단 탭]  캘린더(1) · 거래처(3) · 정산(4) · 단가표(5)

1 캘린더(홈)
   └ 날짜 탭 → 당일 작업 시트
   └ + FAB → 2 작업 기록 추가
3 거래처 목록
   └ 카드 탭 → 5 단가표 편집
   └ + 등록 → 3b 거래처 등록 → (다음) 5 단가표
4 정산 내역
   └ [정산 메시지 만들기] → 4b 모달
5 단가표 편집
```

핵심 진입 동선은 **캘린더 → +FAB → 작업 기록(3탭) → 저장 → 캘린더**. 이 루프의 마찰 최소화가 제품의 전부.

---

## 2. 화면별 스펙

### 화면 1 — 캘린더 (홈)
- **상단 요약(고정):** 이번 달 수익(합계), 미수금(합계 + 14일 경과 건수). 항상 노출.
- **월/주 토글.** 기본 월.
- **달력 셀:** 작업 있는 날은 금액 표기 + dot. 미수 건 있는 날은 경고색 dot.
- **날짜 탭:** 당일 작업 시트 — 각 행 = 거래처·작업명 / 팀원 수·현장·사진 수 / 금액 · 입금상태 배지.
- **+ FAB:** 작업 기록 추가로 이동. 캘린더 화면에서만 노출.
- 상태: 작업 0건인 달 → 빈 상태 카피 "기록을 추가하면 여기에 수익이 쌓입니다."

### 화면 2 — 작업 기록 추가  ★ 핵심
- **필드:** 거래처(필수, select) · 날짜(기본 today) · 작업유형(시공/실측/재단/AS, 기본 시공) · 투입 팀원(필수, multi) · 일수(기본 1) · 사진(선택, 다중) · 메모(선택).
- **자동계산:** 거래처 선택 시 해당 공종 단가 로드 → `금액 = 단가 × 인원 × 일수` 실시간 표시.
- **금액 수기 보정 허용**(추가비·할인 현장). 보정 시 `is_manual=true` 플래그.
- **검증:** 거래처·팀원 1명 이상 필수. 단가 미설정 거래처면 단가 입력 인라인 유도(PM 결정 1).
- **저장 후:** 캘린더 해당 날짜로 복귀, 셀에 즉시 반영.

### 화면 3 — 거래처 목록
- 검색(업체명·지역·공종). 카드 = 업체명 · 유형 배지 · 위치·공종 · **단가 요약**.
- 카드 탭 → 단가표(5). + 등록 → 3b.
- 정렬: 최근 사용순 기본.

### 화면 3b — 거래처 등록
- **필수:** 업체명 · 유형(공장/인테리어/대리점·유통/개인·기타) · 주요 공종(multi).
- **선택:** 위치 · 담당자·연락처.
- **단가 입력 건너뛰기 허용** → "다음" 시 단가표(5)로 이동하되 비워둘 수 있음.

### 화면 4 — 정산 내역
- 월 선택. 상단 요약(총 정산액·미수금).
- **거래처별 / 팀원별 토글.** 거래처별=받을 돈·입금상태, 팀원별=줄 돈·투입일수.
- 미수 건 `미수 N일` 배지. **N ≥ 14 → 알림 트리거.**
- 입금 확인: MVP **수동 체크**(행에서 입금완료 처리).

### 화면 4b — 정산 메시지 (모달)  ★ 킬러
- 선택 거래처(또는 전체)의 해당 월 기록을 모아 **카톡 텍스트 자동 생성**.
- 포맷은 §4 비즈니스 로직 참조. [복사] → 클립보드.

### 화면 5 — 단가표 편집
- 거래처 단위. 행 = `공종 → 일당`. 추가/삭제.
- 저장 시 즉시 반영. **과거 기록의 금액은 변경하지 않음**(스냅샷 보존).

## 3. 데이터 모델

```
Partner (거래처)
  id            uuid
  name          string                 *필수
  type          enum(factory|interior|dealer|personal)  *필수
  location      string?  (lat,lng?)
  trades        string[]               *필수  // 공종
  contact_name  string?                // 앱 내 노출 only
  contact_phone string?
  created_by    user_id
  created_at    datetime

RateTable (거래처별 단가)
  id          uuid
  partner_id  uuid → Partner
  trade       string        // 공종명
  daily_rate  int (KRW)     // 1인 1일 일당
  // (partner_id, trade) unique

Member (팀원, 비가입)
  id        uuid
  owner_id  user_id        // 팀장
  name      string
  phone     string?

WorkLog (작업 기록)  ★ 중심 엔티티
  id            uuid
  owner_id      user_id
  partner_id    uuid → Partner
  date          date          *필수
  work_type     enum(install|measure|cut|as)   default install
  member_ids    uuid[]        *필수 (≥1)
  days          number        default 1
  unit_rate     int           // 기록 시점 단가 스냅샷
  amount        int           // = unit_rate × len(member_ids) × days
  is_manual     bool          // 금액 수기 보정 여부
  photos        url[]         // 선택
  memo          string?
  payment       enum(unpaid|paid)  default unpaid
  paid_at       datetime?
  created_at    datetime
```

- **금액은 기록 시점 단가(`unit_rate`)를 스냅샷**으로 저장 → 이후 단가표를 바꿔도 과거 금액 불변.
- 정산 합계는 `WorkLog` 집계로 도출(별도 정산 테이블 불필요한 MVP).

## 4. 비즈니스 로직

**4-1. 금액 자동계산**
```
amount = unit_rate × member_ids.length × days
```
거래처+작업유형으로 `RateTable.daily_rate` 조회 → `unit_rate`. 수기 보정 시 `is_manual=true`, `unit_rate`는 참고용 유지.

**4-2. 미수금 / 14일 알림**
- 미수금 = `Σ amount where payment=unpaid`.
- 경과일 = `today − date`. `경과일 ≥ 14` → 알림(푸시/배지). 30일 별도 단계 표시 권장.

**4-3. 정산 메시지 자동 생성** ★
- 입력: 거래처(또는 전체) + 월.
- 그룹핑: 날짜 → 현장(거래처) → 팀원별 금액.
- 출력 텍스트 골격:
```
안녕하세요 사장님, {월} 시공비 내역입니다!

*{M/D} {현장}
 {팀원} {금액} / {팀원} {금액} ...
...

합계 {총액}원
입금계좌: {계좌}
```
- **실제 팀장 카톡 양식을 그대로 모사**해야 함 → 베타 기사 샘플 수집 후 포맷 확정(PM 결정 4). 계좌·팀원 표기 방식은 사용자 설정값.

## 5. MVP 체크리스트

- [ ] 캘린더 월/주 + 요약(수익·미수금)
- [ ] 작업 기록 추가 (자동계산 + 수기 보정)
- [ ] 거래처 목록·검색·등록
- [ ] 거래처별 단가표 (스냅샷 저장)
- [ ] 정산 내역 (거래처별/팀원별, 수동 입금체크)
- [ ] 미수금 14일 알림
- [ ] 정산 메시지 자동 생성 + 복사
- [ ] 사진 첨부

**비범위(Phase 2+):** GPS 자동 작업기록 · 통장 자동매칭(오픈뱅킹) · 일감 매칭/공장 계정/블라인드 매칭 · 카톡 정산메시지 역임포트.

## 6. 결정 필요 (PM)

1. 단가 입력 시점 — 등록 필수 vs 첫 기록 때 요청. (권장: 후자)
2. 팀원 — 비가입 라이트 확정 여부.
3. 입금 확인 — 수동 체크 MVP 확정.
4. **정산 메시지 포맷** — 실제 팀장 카톡 샘플 수집 필요(차단 이슈).
5. 거래처 시드 — 보유 공장 5곳 사전 입력 여부.
