# 바로기사(Barogisa) — MVP·데모 개발 총정리

> **문서 목적** 그동안 흩어져 진행된 MVP 개발·데모 제작·개발 환경 구축 관련 의사결정을 한 문서로 통합
> **제품** (주)도어링의 주방가구 시공 운영 SaaS "바로기사"
> **최종 정리** 2026-06-09

---

## 1. 제품 한 줄 정의

> **여러 공장을 오가며 일하는 주방가구 시공기사를 위해 수주·현장기록·청구·수금을 한 흐름으로 자동화하는 운영 SaaS.**

- 시공기사(팀장 중심) → 무료 운영 툴
- 공장·인테리어 업체 → 검증된 시공기사 풀 접근에 10% 거래 수수료
- 블라인드 매칭으로 앱 외부 직접 접촉 차단 → 공장 측 구조적 락인

---

## 2. MVP 핵심 5단계 흐름 (제품의 전부)

```
[공장] 일감 등록
    ↓ 푸시 알림
[기사] 한 탭으로 수락/거절 (일정 충돌 시 자동 경고)
    ↓
[기사] 현장 도착 → GPS 자동 체크인 + 작업 시작 사진
[기사] 작업 완료 → 체크아웃 + 완료 사진
    ↓
[앱] 월말 공장별 작업횟수·금액 자동 합산 → 청구서 PDF 자동 생성
    ↓
[공장] 송금 → [기사] 입금 확인 한 탭 매칭
```

이 5단계가 1차 가치 전부다. 부가 기능(콜센터 버튼, 바로가구 직발주)은 이후 단계.

---

## 3. MVP 범위

### 3.1 포함 (이번에 만드는 것)

핵심 화면 5개를 우선순위 순으로 만든다.

| 순위 | 화면 | 설명 |
|------|------|------|
| 1 | **캘린더 화면** | 시공기사 메인. 본인 일정(파랑) + 팀원 일정(초록) + 신규 일감(빨강) 색상 점. 상단에 이번달 수익 / 미수금 / 이번주 일감 요약 3칸 |
| 2 | **일감 수락/거절 카드** | 공장명·날짜·공정·단가 표시. 한 탭 수락/거절 |
| 3 | **체크인/체크아웃** | GPS 자동 인식 + 사진 첨부 |
| 4 | **정산 화면** | 공장별 작업 내역, 미수금 추적, 청구서 PDF 생성 |
| 5 | **공장 측 일감 등록** | 일자·공정·주소·지정기사 입력. 단가 자동 입력(시장 표준) |

### 3.2 제외 (지금 만들지 않는 것)

- 결제·정산 자동화 → 수기 확인으로 우회
- 보증금 실제 입금 처리 → UI 플레이스홀더만
- 별점·전문성 인증 시스템 → 2단계
- 추천 알고리즘 → 1단계 이후
- 아카데미/교육 기능 → 3단계
- 세금계산서·부가세 연동

## 4. 캘린더 양방향 매칭 (앱의 심장) — 상세 설계

캘린더는 단순 일정표가 아니라 "기사 주도(구직)"와 "공장 주도(구인)" 두 모드가 같은 화면에서 공존하는 매칭 엔진이다.

### 4.1 두 모드

**A. 기사 주도 (구직)**
```
기사가 "○월 ○일 가능" 등록 → 공장이 보고 예약 신청 → 기사 수락 → 양쪽 캘린더 확정
```

**B. 공장 주도 (구인)**
```
공장이 일감 공고 등록 → 조건 매칭 기사에게 자동 알림 → 기사 지원 또는 공장 직접 지목 → 공장 선택 → 양쪽 캘린더 확정
```

### 4.2 슬롯 상태 5가지 (신호등 색상 체계)

| 색상 | 상태 |
|------|------|
| 회색 | 비어있음 |
| 초록 | 가능표시함 (공장에게 노출 중) |
| 노랑(주황) | 예약신청 받음 / 기사 수락했으나 공장 미확정 |
| 파랑 | 확정됨 |
| 검정 | 완료됨 |

### 4.3 충돌·동시신청 처리

- 한 기사에게 여러 공장이 동시 신청 가능
- 기사가 하나 수락 → 동시간대 다른 신청 자동 거절 + 해당 공장 알림
- 확정 시 그 슬롯 가능표시 자동 해제
- 공장은 동일 슬롯에 여러 기사 확정 가능 (인원수 채울 때까지)

### 4.4 일감 단위 채팅 (필수)

- 예약 신청/지원 발생 시 1:1 채팅방 자동 생성
- 상단에 일감 정보 카드 고정 (날짜·주소·단가)
- 확정 후에도 "현장 도착 알림", "마무리 사진 공유" 용도로 계속 사용
- MVP 구현: 폴링 기반(5초 간격), 읽음 표시 단순화

## 5. 데이터 모델 (핵심 엔티티)

```
User: id, role(복수 가능), phone, name, profileImage, regions[], referralCode, referredBy, deposit_status
Profile: tags[], certifiedTags[], bio, rating, completedJobs
Availability: userId, date, timeSlot, status(available/booked)
Job: requesterId, date, timeSlot, region, requiredTags[], headcount, pay, memo, status
JobApplication: jobId, workerId, status(pending/accepted/rejected/completed)
Review: jobId, reviewerId, targetId, rating, comment
Referral: referrerId, refereeId, totalPayback
CertificationRequest: userId, tag, evidenceUrl, status
```

> 인력 카테고리는 enum이 아니라 **다중 태그**로 설계 (블럼·헤펠레·세라믹 등). 향후 "한식당 전용 시공" 같은 태그 확장 용이.


```typescript
const STANDARD_RATES = {
```

## 9. 핵심 비즈니스 규칙

- 공장은 기사를 **지명**해서 발주한다 (익명 매칭 없음)
- 양측 수락 후 거래 성립 (공장 발주 → 기사 수락 → 공장 최종 확정)
- 주황 상태 = 기사가 수락했으나 공장이 아직 미확정
- 공장 연락처는 **앱 내에서만 노출** (탈중개 방지, 락인 핵심)
- 팀장이 팀원 등록 시 페이백 지급 (바이럴 유입 전략)

## 10. 개발 시 주의사항 (UX 원칙)

- 모바일 퍼스트. 기사는 현장에서 한 손으로 조작 → 큰 터치 영역, 큰 글자
- 핵심 액션(수락/거절/체크인)은 **한 탭**으로 완료
- GPS 체크인 마찰 최소화가 핵심 UX 과제
- 오프라인에서도 체크인 기록 저장 (현장 인터넷 불안정)
- 복잡한 추상화보다 창업자가 직접 검토 가능한 명확한 코드 우선
- 한국어 UI (모든 라벨/메시지)

## 12. 데모/프로토타입 제작 이력

### 12.1 현장픽 (HyunjangPick) — 초기 웹 데모
- 산업/작업복 미감의 주방시공 인력 매칭 데모
- Next.js + Prisma + SQLite 스택 기반 PRD(spec.md) 작성
- 캘린더 양방향 매칭, 전문성 태그, 팀장 페이백 중심
- → 이후 제품 방향이 모바일 운영 SaaS로 정리되며 네이밍·스택 변경

### 12.2 바로기사 HTML 클릭더미 — IR 시연용
- **5개 핵심 화면** 작동 데모 (모바일 프레임 + 클릭 흐름)
  1. 공장 — 일감 등록
  2. 기사 — 일감 목록·수락
  3. 기사 — 현장 체크인
  4. 기사 — 미수금 대시보드
  5. 공장 — 미지급 대시보드
- 인증·DB 없는 클릭더미, VC 미팅 노트북 시연용
- 다크 네이비(#0F1B2D) + 앰버(#F59E0B) 톤

### 12.3 SVG/HTML 목업 — 발표·IR 자료용
- 캘린더 중심 UI + 신호등 시스템(빨강=확정 예약, 주황=공장 미확정, 초록=가능)
- 정산 대시보드: 자동 은행 대사 + 미수금 추적
- HTML 위젯 → SVG 파일 형태로 발전

### 데모 단계 전략 정리

- **1단계 (지금):** 클릭더미 프로토타입 — Claude Code로 창업자 직접 제작 가능
- **2단계 (시드 후 6개월):** 실제 베타 MVP — 개발자 채용 + Claude Code 보조

> 핵심 원칙: 베타 사용자가 쓰는 MVP는 시드 자금으로 만드는 것이지, 받기 전에 만드는 것이 아니다. 시드 전 창업자가 만들 수 있는 현실적 범위는 클릭더미까지.
